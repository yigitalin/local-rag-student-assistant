# Domain Mapping

## Purpose

The purpose of this document is to map the original Microsoft Gas Field Support Agent knowledge base to the selected Oil Refinery Equipment Support Agent domain.

Rather than redesigning the project, this document defines how each document, equipment type, and technical concept will be adapted while preserving the original Microsoft Local RAG architecture and document structure.

The goal is to replace the application domain with minimal modifications, ensuring that the overall RAG workflow, software architecture, and knowledge organization remain unchanged.

## Mapping Principles

During the adaptation process, the following principles will be followed:

- Preserve the original Markdown document structure.
- Preserve the original document hierarchy.
- Preserve the overall RAG workflow.
- Preserve the existing software architecture.
- Replace domain-specific terminology where appropriate.
- Avoid unnecessary document restructuring.
- Maintain the same level of technical detail as the original project.

## Mapping Strategy

Each Microsoft Markdown document will be analyzed individually.

For every document, we will identify:

- The original purpose of the document.
- Its equivalent role in the Oil Refinery domain.
- Required terminology replacements.
- Required content updates.
- Whether the document structure can remain unchanged.

This mapping process ensures that every adaptation decision is documented before implementation.
